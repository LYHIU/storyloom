## 一、AI 基础设施

### 配置 (`src-tauri/src/commands/ai.rs`)

- **AiConfig** 结构：`enabled`, `provider`, `base_url`, `api_key`, `model`, `verified`
- 配置文件路径：`{vault}/ai_config.json`
- `verified` 数组记录测试连接成功的 provider，前端快速切换只显示已认证的

### 命令
- `get_ai_config(vault_path)` — 读取配置
- `save_ai_config(vault_path, config)` — 保存配置
- `ai_chat(vault_path, request)` — 发送对话请求

### 双通道
- **OpenAI 兼容** (DeepSeek/OpenAI/Claude)：POST `{base_url}/chat/completions`
- **Ollama 原生** (provider == "ollama")：POST `{base_url}/api/chat`，响应解析 `message.content`

### 前端
- `AiSettings.tsx`：弹窗式设置面板，toggle 开关，四选一 provider，测试连接，管理已认证列表
- `NameWorkshop.tsx`：快速模型切换胶囊按钮，只显示 verified 的 provider

### 依赖
- Rust: `reqwest 0.12` (json), `tokio 1` (full)
- 前端: 无额外部依赖，走 `ai_chat` Tauri command

## 二、起名工坊 (`src/scenes/ideation/NameWorkshop.tsx`)

### 分类（8 种）
角色名 / 地名 / 功法技能 / 帮派组织 / 武器法宝 / 称号昵称 / 书名 / 章节名

### 参数
- 性别（仅角色名）：男 / 女 / 不限
- 语言：中文 / English（切换 prompt 和 system message）
- 风格下拉：古风、现代、民国、西幻、仙侠、科幻、都市、不限
- 数量：5 / 10 / 15 / 20
- 额外要求：自由文本输入

### AI 调用
- System message 根据语言切换，中文版强制"只输出纯中文汉字名字"
- 额外要求含拼音首字母时自动补充"请取拼音首字母满足条件的中文名字"
- 解析：按行分割 → 去序号前缀 → 过滤空行、过长(>20字)、括号开头
- 防重复点击：`loading` 状态 + `genIdRef` 序号防旧结果覆盖
- 中止功能：生成中按钮变橙色「⏹ 停止生成」，序号递增跳过旧响应

### 交互
- 结果网格，点选收藏（星星标记）
- 已收藏列表，可清空
- 快速模型切换条（仅显示 verified 的 provider）

## 三、人设卡片 (`src/scenes/ideation/CharacterCards.tsx`)

### 数据模型 (`src/lib/tauri.ts`)

```typescript
CharacterBoard {
  version: 1
  characters: CharacterCard[]
  relationships: CharacterRelationship[]
  conflicts: ConflictSeed[]
}

CharacterCard {
  id, name, aliases, role (protagonist|supporting|antagonist|minor)
  color, importance (1-5)
  fields: CharacterField[]  // 可自定义
  chapterIds: string[]
}

CharacterField { id, label, value, kind: 'short'|'long' }

CharacterRelationship {
  id, fromId, toId, type
  surface, truth, tension, power, emotion
  stage (相识→决裂 8 段), notes
}

ConflictSeed {
  id, title, characterIds[], relationshipIds[]
  premise, pressure, possibleTurn, reverseQuestion
  status: 'seed'|'used'|'archived'
}
```

### 存储
- Rust 命令：`read_character_board` / `save_character_board`
- 文件：`{project}/characters.json`
- 自动保存：600ms 防抖，dirty flag

### 布局（三栏）
- **左栏 (260px)**：角色列表 + 搜索 + 角色定位筛选
- **中栏（自适应）**：角色编辑器（姓名、别名、角色定位、颜色、重要度、字段卡片、出场章节）
- **右栏 (380px)**：关系 / 冲突面板 + 列表/图谱切换

### AI 分析
- `runAiAnalysis(prompt)` 调用 `aiChat`，使用 `vaultPath` 读配置
- 每个关系卡片有「AI 分析」按钮，拼装角色字段作为上下文
- 每个冲突卡片有「AI 分析」按钮
- 结果显示为可关闭紫色提示卡片
- System message：小说编辑顾问角色

### 关系图谱
- `reactflow` 实现（`RelationshipGraph` 组件）
- 节点：角色色块，显示姓名
- 边：箭头连线，标签显示关系类型和阶段
- 布局：4 列网格
- 功能：缩放、拖拽、缩略图 (MiniMap)、Controls

### 字段管理
- 默认 10 字段：表层身份、外貌、性格、欲望、恐惧、软肋/底线、秘密、剧情功能、成长弧线、当前状态
- 用户可增删字段
- short 字段用 input，long 字段用 textarea
