# 书织 (StoryLoom) — 产品设计规范

## 概览

- **产品名称**：书织 (StoryLoom)
- **产品概念**：书织回文锦，墨通山海间。以字为经、以章为纬，陪用户织起人物、情节与世界。当梭声停下，万象俱全。
- **产品形态**：桌面可执行程序（Windows exe）
- **技术栈**：Tauri (Rust) + React + TypeScript + Tailwind CSS
- **数据存储**：本地 Markdown + JSON，零云端依赖
- **目标用户**：业余→职业网文写手，耽美等细分风格作者
- **核心差异**：场景化写作空间 + AI 只辅助不生成

---

## 一、四大场景模式

产品按写作流程分为四个独立场景，顶部标签栏切换，每个场景拥有专属界面布局和工具集。

### 1. 构思空间

目的：从零到有，灵感→人设→大纲。

默认背景：淡茶米 `#f6f3ed`（暖调激发灵感）。

**起名工坊**
- 角色名/书名/章节名/地名/功法名分类生成
- 可选参数：性别、性格、风格（古风/现代/民国/西幻等）、字数、首字母
- AI 基于用户设定生成候选名字列表，不生成其他内容
- 满意的名字可收藏、分类、比较
- 撞色点缀：雀黄 `#e8c560`

**情节图谱**
- 两种模式：自由绘制（空白画布 + 节点连线）和 AI 对话式（基于用户自己设定给建议）
- 节点类型：冲突点 / 情感转折 / 人物出场 / 高潮 / 收束
- 不内置任何内容模板（无废材逆袭、退婚流等预设套路）
- 内置通用叙事结构框架（三幕、五幕、起承转合），用户自行填充
- AI 辅助：卡节点时给出「接下来可能发生什么」的多个方向选项

**人设卡片**
- 定位：不是单纯人物资料库，而是「人物驱动剧情」工作台。
- 第一版结构：左侧角色列表，中间档案卡编辑，右侧关系与冲突种子面板。
- 角色档案：结构化人物档案，字段自由增减。
- 默认字段：表层身份、外貌、性格、欲望、恐惧、软肋/底线、秘密、剧情功能、成长弧线、当前状态。
- 用户可添加自定义字段（如 CP 关系、攻受属性、情感弧线、能力体系、禁忌点等）。
- 角色关系：关系作为独立「边」存储，记录表面关系、真实关系、张力来源、权力差/筹码、情感温度、关系阶段与备忘。
- 冲突种子：从人物欲望、恐惧、秘密或关系张力提炼剧情素材；支持正推「人物/关系 → 矛盾冲突 → 情节节点」，也支持倒推「既定剧情 → 补足人物动机/秘密/代价」。
- 当前存档：项目目录下 `characters.json`，包含 characters、relationships、conflicts 三类数据。
- 后续演进：关系图谱自动生成，冲突种子与情节图谱节点互相引用，正文悬停人名弹出人设摘要。
- AI 辅助：人物一致性检查（跨章节行为是否与设定矛盾），只做检查和提示，不代写人设。

**世界观画布**
- 无限画布式构建世界观
- 支持图片参考、文字说明、关系连线
- 预设结构框架（修仙体系/魔法体系/王朝体系等），用户自行填充内容

### 2. 码字空间

目的：专注产出，大纲→正文。

默认背景：宣纸白 `#fafaf7` + 编辑器纸色 `#fffef9`。

**布局**
- 左侧：大纲侧栏（可收起），卷→章树形结构，拖拽排序
- 右侧：编辑器主体，占据主要面积
- 底部：状态栏（今日字数/目标/连续天数/小黑屋计时）

**编辑器**
- 纯文本编辑，支持 Markdown 轻量排版
- 字体/字号/行距/主题色完全可调
- 打字机模式（当前行居中）+ 专注模式（全屏，UI 淡出）
- 自动保存 + 手动快照版本

**小黑屋**
- 设定目标字数或时长，锁死界面直到完成
- 可调严厉程度：仅提醒 → 禁止切场景 → 禁止切应用
- 撞色点缀：蜜柑橙 `#f0a060`

**大纲侧栏**
- 卷→章树形结构，拖拽调整顺序
- 每章显示字数、状态（草稿/完成/待修改）
- 点击跳转，搜索章节名

**快速查阅**
- 侧边浮窗：悬停人名/地名弹出入设卡片摘要

**AI 辅助（可完全关闭，关闭后进入纯净小黑屋）**
- 按快捷键唤起 AI 输入框
- 描述需要的词或表达，AI 返回词/短语列表
- 系统约束：只输出词汇/短语选项，禁止输出完整句段
- AI 功能有独立开关，关闭后所有场景无 AI 干扰

### 3. 润色空间

目的：打磨提升，草稿→终稿。

默认背景：竹青白 `#f2f6f0`（清爽审视）。

**布局**
- 左侧：章节原文（可编辑）
- 右侧：分析面板（工具叠加）

**工具面板**
- 节奏热力图：段落级着色（红=动作密集 / 黄=过渡 / 蓝=抒情描写）
- 重复用词检测：短距离内重复词/短语高亮，支持忽略白名单
- 语法纠错：搭配不当、成分残缺、语序问题（AI 驱动，仅建议不强制）
- 用词检查：语境下用词是否恰当（如古风背景出现现代口语）
- 标点检查：中英文标点混用、引号不配对
- 人物一致性：同一人物名字/特征前后不一致
- 段落字数分布：可视化每段字数，提示手机端阅读体验
- 撞色点缀：樱紫 `#b895b0`（标识 AI 纠错建议）

**用户词库**
- 纯文本管理界面，用户自行增删改
- 初始为空，不预设任何词
- 支持分组（按平台/类型），润色时按组勾选

### 4. 分析空间

目的：用数据认识自己的写作模式。

默认背景：淡茶米 `#f6f3ed`。

**码字日历**
- 热力图风格（类似 GitHub 贡献图）
- 每天字数颜色深浅表示，点击某天看当天写了哪些章节
- 连续码字天数、本月总量、历史最高单日

**时段分析**
- 基于写作速度和修改频率自动判断最佳创作时段
- 识别常卡文时段

**全书进度**
- 计划总字数 vs 已完成字数
- 预估完本日期（基于近期平均速度）
- 各卷/章字数分布饼图

**卡文记录**
- 写作时可手动标记「卡住了」
- 回顾：哪些章节卡了？卡了多久？如何突破？
- 形成个人卡文模式记录

**作品时间线**
- 纵向时间线展示从第一章到最新一章的创作历程
- 标注大改、重写、断更和恢复节点
- 撞色点缀：天水蓝 `#7db8c4`

## 三、AI 设计

### 核心原则

AI 是写作工具，不是写作作者。禁止生成任何正文内容。

| 场景 | AI 能做 | AI 不能做 |
|------|---------|-----------|
| 构思空间 | 根据用户设定生成名字候选；提示情节缺口 | 写人设、编情节 |
| 码字空间 | 根据描述给出词/短语候选 | 写句子、续写段落 |
| 润色空间 | 语法纠错、用词不当提示、一致性检查 | 改写、优化修辞 |
| 分析空间 | 统计和可视化 | 评价作品质量 |

### 技术实现

- **混合模式**：轻量任务（起名、查词）→ 本地 Ollama；重量任务（纠错、一致性）→ 云端 API
- **降级策略**：云端不可用时降级到本地模型
- **API 兼容**：OpenAI 兼容格式，支持 DeepSeek/Claude/GPT 切换
- **全局开关**：AI 功能可一键关闭，关闭后所有场景无 AI 干扰
- **用户自配**：用户自行配置 API Key 和模型选择

## 四、技术架构

### 分层

```
Web 前端 (React + TypeScript + Tailwind)
    ↓ Tauri invoke / Event
Tauri Rust 后端
    ├── 文件管理（读写、版本、导入导出）
    ├── AI 调度器（本地/云端路由分发）
    ├── 数据引擎（SQLite、全文搜索）
    └── 系统服务（自动更新、系统托盘、全局热键）
```

### 前端

- React + TypeScript + Tailwind CSS
- 关键依赖方向：dnd-kit（拖拽）、react-flow（图谱/画布）、tiptap 或 lexical（编辑器）

### 数据存储

- 正文：Markdown 文件，用户指定存储目录
- 当前元数据：本地 JSON（`project.json` 存章节顺序；`characters.json` 存角色档案、角色关系、冲突种子）
- 后续元数据：可迁移到 SQLite（章节索引、码字记录、人设卡片、AI 对话历史）
- 备份：一键打包 zip + 可选定时自动备份
- 全部本地，零云端依赖

### 前端目录结构

```
src/
├── scenes/          # 四大场景
│   ├── ideation/    # 构思空间
│   ├── writing/     # 码字空间
│   ├── polishing/   # 润色空间
│   └── analytics/   # 分析空间
├── components/      # 共享组件
├── stores/         # 状态管理
├── lib/            # 工具函数、AI 调用封装
└── styles/         # 主题系统
```

## 六、像素伙伴（Pixel Buddy）

灵感源自"数织"（Nonogram）填色游戏，与产品名"书织"一字相扣。

### 核心概念

每本新书在创建时，程序随机生成一个独一无二的 16×16 像素小生灵（类似拓麻歌子的抽象形体，非特定动物物种）。用户在写作过程中，通过持续码字逐格点亮像素，逐步看到伙伴的完整样貌和日常活动。

### 新建作品配置

用户在新建作品时设定：
- **预计总字数**：输入框，预估全书字数
- **伙伴成长速度**：滑块（快→慢）
  - 往左（快）：更少字填一格，伙伴成长快，画面变化频繁，但长篇小说后期活动场景会重复
  - 往右（慢，默认推荐）：更多字填一格，16 种活动场景刚好覆盖全书，写完最后一字出完最后一幅，零重复

### 计算逻辑

- 1 张像素伙伴图 = 16×16 = 256 格，全部彩色填充，无空白格
- 全书总格数 = 16 张活动 × 256 格 = 4,096 格
- 每格字数 = 预计总字数 ÷ 4,096
- 默认推荐（最慢位置）：10 万字 → 约 24 字/格，全书出完 16 张，刚好写满不重复
- 滑块最左（最快）：10 万字 → 约 12 字/格→ 全书出 32+ 张，16 种活动轮完后再循环

### 伙伴造型

- 新建作品时程序随机生成像素形体 + 3-4 种配色
- 全书范围内造型完全一致，不变轮廓不变色
- 不同作品之间造型和配色各不相同

### 写作过程中的显示

- 编辑器侧栏底部显示 16×16 像素网格（约 256px 见方）
- 每写够 N 字，一个格子从灰暗变为彩色
- 第一章过程中，伙伴形体逐渐浮现 → 格子全部亮起时首次完整亮相
- 后续每完成一幅新活动图，网格重置，开始解锁下一幅

### 16 种预设活动场景

进食、打盹、玩耍、奔跑、跳跃、伸懒腰、打哈欠、躲猫猫、观察、探险、发愣、打滚、攀爬、游泳、晒太阳、眺望

每幅场景中，伙伴造型不变，背景和动作变化。

### 性格标签

基于写作内容特征自动生成伙伴的性格描述：

| 内容特征 | 性格标签 |
|---------|---------|
| 对话比例高 | 活泼好动，喜欢听键盘声 |
| 动作描写多、场景切换快 | 精神十足，总在桌上蹦蹦跳跳 |
| 心理描写多、抒情段长 | 安静温顺，爱趴在手边发呆 |
| 古风词汇丰富 | 端庄矜持，走路带着书卷气 |
| 冲突/战斗密集 | 胆子很大，竖着尾巴看你的文字 |
| 甜宠/温馨内容多 | 软乎乎的，最爱被摸头 |

共 12-16 组映射规则，每完成几章后更新一次标签。

### 完本图鉴

分析空间展示已完本书籍的像素伙伴墙——每本书的伙伴造型、全 16 幅活动场景缩略图、性格描述一览。

### Phase 1：MVP（骨架可用）
- Tauri + React 项目搭建
- 码字空间（编辑器 + 大纲侧栏 + 小黑屋 + 本地存储）
- 基础文件管理（创建/打开作品、Markdown 读写）
- 主题系统（竹林清风配色）

### Phase 2：构思 + 润色
- 构思空间（起名工坊 + 人设卡片第一版已接入；情节图谱 + 世界观画布待接入）
- 润色空间（AI 纠错 + 重复检测 + 节奏热力图 + 用户词库）
- AI 调度器（Ollama 本地 + 云端 API 混合）

### Phase 2b：像素伙伴（数织灵感）
- 新建作品时随机生成一个独一无二的 16×16 像素小生灵（类似拓麻歌子的抽象形体，非特定动物）
- 用户设定总字数 + 成长速度（滑块），系统计算每格字数
- 写作过程中，编辑器侧栏显示 16×16 像素网格，每写够 N 字点亮一格
- 第一章格子全部亮起时，伙伴造型首次完整定型
- 全书共预设 16 种活动场景（进食、打盹、玩耍、奔跑、跳跃、伸懒腰、打哈欠、躲猫猫、观察、探险、发愣、打滚、攀爬、游泳、晒太阳），按顺序分配
- 伙伴造型全书画一，变的是场景背景和动作
- 每本书的伙伴独一无二，不同作品之间不重复
- 伙伴性格描述基于写作内容特征生成（对话比例、心理描写、动作密度等映射到性格标签）
- 像素网格设计为彩色填充，16×16=256 格全部参与渲染，无一格浪费
- 完本后在分析空间展示伙伴图鉴墙

### Phase 3：分析 + 打磨
- 分析空间（码字日历 + 时段分析 + 进度追踪 + 作品时间线 + 像素伙伴图鉴）
- 全文搜索、快照版本、导入导出
